WB 运营分析判断体系思维导图

目标：把 WB API、数据库、后端、前端、运营人员和 AI 大模型连成一套判断链路，让运营快速回答：哪里异常、为什么异常、谁处理、下一步做什么。
WB 数据源 数据库层 运营协作 AI 分析 前端判断 风险边界

运营每天先看什么

  • 销售额、订单、转化率是否异常
  • 广告费是否上升但订单没上升
  • 库存是否影响销售
  • 客服/差评/退货是否集中到某个产品

系统必须给什么证据

  • 指标来自哪个 API 和哪张表
  • 按店铺、商品、日期可追溯
  • 统一卢布和原币种同时可看
  • 异常原因可下钻到原始记录

AI 应该怎么帮忙

  • 把异常排序成今日重点
  • 说明最可能原因和证据
  • 给下一步动作和负责人
  • 生成日报、周报和复盘

人工必须决定什么

  • 是否改价格、改预算、改商品卡
  • 是否参加促销或补货
  • 是否处理异常费用
  • 是否回复敏感客服问题

12 类数据域：流程、人员、数据支持、AI 分析

1. 店铺 / 账号域

权限、主体、评分、token 状态

流程

检查 token 权限 -> 同步主体信息 -> 标记可用模块 -> 监控过期和权限缺口。

参与人员

系统管理员、店铺负责人、技术维护。

数据支持

seller_id、supplier_id、权限类目、只读状态、评分、评价数量。

AI 分析

解释某店铺为什么缺销售/广告/客服数据，输出权限补齐清单。

2. 商品内容域

商品卡、图片、标题、属性、类目

流程

同步商品卡 -> 建商品维表 -> 归入产品组 -> 和销售/广告/客服关联。

参与人员

商品运营、设计、翻译、负责人。

数据支持

nmId、imtID、barcode、vendorCode、brand、subject、title、description、media。

AI 分析

发现标题、主图、属性缺陷；结合转化率给商品卡优化建议。

3. 价格 / 折扣域

价格、折扣、促销、上传状态

流程

同步价格与促销 -> 记录价格历史 -> 结合销量和利润判断价格效果。

参与人员

运营、财务、负责人。

数据支持

price、discounted_price、discount、club_discount、promo_id、promo_period、upload_status。

AI 分析

判断降价后销量是否改善、促销是否亏损、价格上传是否失败。

4. 库存 / 仓库域

库存、仓库、缺货、在途

流程

同步库存 -> 按仓库和 SKU 聚合 -> 和销量速度计算可售天数。

参与人员

仓库、采购、运营、负责人。

数据支持

warehouse、barcode、quantity、in_way、reserved、stock_days、office。

AI 分析

预警缺货、滞销、断货导致销售下滑，给补货优先级。

5. 订单 / 销售域

运营订单、销售、退货、价格口径

流程

同步订单销售 -> 按 srid 去重 -> 区分订单、销售、取消、退货。

参与人员

运营、负责人、财务。

数据支持

srid、nmId、barcode、date、finishedPrice、forPay、isCancel、saleID。

AI 分析

解释销售变化，识别退货放大、取消异常、区域或仓库异常。

6. Analytics 漏斗域

曝光、点击、加购、订单、转化

流程

按商品和日期同步漏斗 -> 计算点击率、加购率、转化率 -> 进入销售看板。

参与人员

运营、商品负责人、广告投手。

数据支持

openCard、clicks、cartCount、ordersCount、buyouts、conversion、search_position。

AI 分析

定位曝光少、点击低、加购低、转化低分别是哪一层问题。

7. 广告 Promotion 域

活动、预算、关键词、搜索簇、广告效果

流程

同步活动和统计 -> 按 advert_id + nmId + date 归因 -> 对比销售漏斗。

参与人员

广告投手、运营、负责人。

数据支持

advertId、status、budget、cost、views、clicks、ctr、cpc、atbs、orders、keywords。

AI 分析

找出烧钱无订单、关键词浪费、预算错配、广告带动不足。

8. FBS Marketplace 域

组装订单、贴纸、发货单、状态

流程

同步待发订单 -> 生成/跟踪发货单 -> 监控超时和取消。

参与人员

仓库、客服、运营。

数据支持

order_id、status、sticker、delivery_note、warehouse、customer_info、metadata。

AI 分析

预警发货延迟、订单状态异常、可能影响差评和取消的节点。

9. FBW Supplies 域

供货、包装、仓库接收、流向

流程

同步供货单 -> 跟踪包装和仓库 -> 关联库存补充。

参与人员

采购、仓库、物流、负责人。

数据支持

supply_id、warehouse_id、box_type、items、quantity、transit_direction、status。

AI 分析

判断供货是否跟得上销售速度，提示补货优先级和仓库风险。

10. 问答 / 评价域

客户问题、评价、回复、置顶评价

流程

同步问答评价 -> 统一客服 item -> 标记处理状态 -> 回传产品问题标签。

参与人员

客服、商品运营、负责人。

数据支持

question_id、feedback_id、nmId、text、rating、answer、createdDate、state。

AI 分析

总结差评原因、FAQ 缺口、商品说明问题，生成回复草稿和产品改进建议。

11. 聊天 / 退货域

买家聊天、退货申请、处理时效

流程

同步聊天和退货 -> 聚合到客服工作台 -> 关联产品和订单。

参与人员

客服、售后、仓库、负责人。

数据支持

chat_id、message_id、claim_id、reason、photos、status、deadline、processed_at。

AI 分析

识别退货原因集中点、处理超时风险、质量问题和物流问题。

12. 财务 / 对账域

结算、费用、扣费、利润、单据

流程

同步 realization report -> 拆费用项 -> 对齐销售/广告/仓储 -> 计算利润。

参与人员

财务、负责人、运营。

数据支持

report_id、sale_dt、for_pay、commission、storage_fee、penalty、logistics、currency。

AI 分析

解释利润变化、异常扣费、广告费侵蚀利润，输出经营真实情况。

AI 汇总后的运营分析输出

今日经营摘要

  • 销售额、订单、广告费、利润变化
  • 统一卢布 / RUB / CNY 分口径说明
  • 最值得关注的 3 个产品和 3 个店铺

异常归因

  • 销售下降：曝光、点击、加购、转化、库存、价格逐层定位
  • 广告异常：花费、点击、加购、订单、关键词逐层定位
  • 客服异常：差评、退货、聊天投诉反向影响商品判断

行动计划

  • 运营：调价格、促销、商品卡、产品优先级
  • 广告：降预算、加词、停无效活动、查搜索簇
  • 客服/仓库/财务：处理差评、补货、核对异常扣费

落地顺序

第一阶段：统一底层口径

raw_jsonshop_id + nmIdproduct_group币种口径

  • 不先切前端，先双写 raw/fact。
  • 把商品漏斗从广告表里拆出来。

第二阶段：建立运营 View

view_ops_product_dailyview_ops_ad_dailyview_ops_customer_signals

  • Dashboard、产品明细、广告分析共用同一口径。
  • 每个指标可追溯到来源 API 和原始记录。

第三阶段：上线 AI 运营助手

只读分析证据链负责人行动建议

  • 先做日报、异常归因、单品诊断。
  • 高风险动作只给建议，不自动执行。
