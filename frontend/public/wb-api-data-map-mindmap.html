从官方 API 到同步服务、数据库分层、后端接口、前端页面的系统化重设计视图
WB API 来源 同步/清洗 数据库层 后端视图/API 前端/业务页面 当前缺口

核心变化

从“业务表直接混合存结果”改成“raw 原始层、facts 事实层、views 展示层”，每个指标都能追溯到 WB API 来源。

当前最大风险

`ad_records` 混合广告统计和商品漏斗，`orders` 不是完整 WB 原始订单，`products.nm_id` 全局唯一不适合多店铺。

建议路线

先新增数据层并双写，对比新旧口径稳定后，再逐步切换 Dashboard、产品销售明细和广告分析。
